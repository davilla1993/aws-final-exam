Contexte du Projet

Ce projet a été réalisé dans le cadre du cours Introduction à AWS. L'objectif est de concevoir et déployer une architecture serverless sur AWS capable d'ingérer en temps réel les flux de données envoyés par des capteurs IoT industriels via des requêtes HTTP POST.

En parallèle, un site de documentation technique (cette page) est hébergé dans un bucket S3 privé, rendu accessible de manière sécurisée via Amazon CloudFront et le mécanisme OAC (Origin Access Control).

L'intégralité de l'infrastructure est automatisée avec AWS SAM (Serverless Application Model) — aucune ressource n'est créée manuellement.

Architecture Globale

La solution repose sur deux sous-systèmes indépendants :

Sous-système 1 — Pipeline d'Ingestion IoT

Client IoT / test_client.py
       │ HTTP POST
       ▼
Amazon CloudFront (Distribution Ingestion)
       │ HTTPS – CachingDisabled
       ▼
Amazon API Gateway HTTP API
       │ Route POST /
       ▼
AWS Lambda Python 3.11 (index.handler)
       ├──▶ Amazon S3 Data Lake
       │      raw-zone/year=YYYY/month=MM/day=DD/
       └──▶ Amazon DynamoDB (Feature Store)
             request_id | timestamp | s3_path | avg_temp | error_count

Sous-système 2 — Documentation Technique

Navigateur
       │ HTTPS
       ▼
Amazon CloudFront (Distribution Documentation)
       │ OAC SigV4 – CachingOptimized
       ▼
Amazon S3 (Bucket privé – accès public bloqué)
       └── index.html (cette page)

Services AWS Utilisés

☁️

AWS SAM / CloudFormation

Infrastructure as Code. Déploiement automatisé de l'ensemble des ressources via sam deploy.

⚡

AWS Lambda

Fonction serverless Python 3.11. Traitement à la volée des données IoT sans serveur à gérer.

🔗

Amazon API Gateway

HTTP API (v2). Point d'entrée REST exposant la route POST / vers Lambda.

🌐

Amazon CloudFront

CDN mondial. Deux distributions : une devant API Gateway (ingestion), une devant S3 (documentation).

🗄️

Amazon S3

Data Lake brut (partitionnement temporel) et hébergement sécurisé de la documentation.

📊

Amazon DynamoDB

Feature Store analytique. Stockage des métriques agrégées en mode PAY_PER_REQUEST.

🔐

AWS IAM

Rôle Lambda avec permissions minimales : s3:PutObject, dynamodb:PutItem, logs CloudWatch.

📋

Amazon CloudWatch

Supervision et débogage. Logs structurés de chaque exécution Lambda (succès et erreurs).

Fonctionnement du Pipeline

ÉtapeActionRésultat
1Client envoie un POST vers CloudFrontRequête HTTPS sécurisée
2CloudFront transmet à API GatewayCache désactivé (POST)
3API Gateway invoque LambdaPayload JSON transmis
4Lambda valide le payload400 si invalide
5Lambda calcule les métriquesavg_temperature, error_count
6Lambda écrit dans S3Fichier JSON partitionné
7Lambda écrit dans DynamoDBLigne avec request_id unique
8Lambda retourne la réponse201 Created

Commandes de Déploiement

Déploiement de l'infrastructure

sam deploy \
  --template-file infrastructure/template.yaml \
  --stack-name fgbossou-iot-stack \
  --capabilities CAPABILITY_NAMED_IAM \
  --resolve-s3 \
  --region eu-west-3

Upload de la documentation vers S3

aws s3 cp index.html \
  s3://629193321657-fgbossou-iot-pipeline-tech-doc/index.html \
  --region eu-west-3
